# SmartStadium Pulse OS — API Documentation (v1.2)

The SmartStadium Pulse OS API provides real-time crowd telemetry, AI insights, and operational controls. 

**Base URL:** `http://localhost:3001/api` (Local) or your Cloud Run service URL.

---

## 🔐 Authentication
Administrative endpoints require an `X-Admin-Key` header matching the `ADMIN_API_KEY` configured in the server environment.

---

## 📡 REST Endpoints

### 🟢 Public Data

#### `GET /health`
Deep health check of the backend engine. Returns memory usage, tick count, and connected clients.

#### `GET /heatmap`
Returns live density data for all 8 zones, including X/Y coordinates for map rendering.
*   **Caching:** 2-second in-memory cache.

#### `GET /metrics`
Returns derived operational KPIs (Congestion Reduced, Wait Time Reduced, Satisfaction Score).
*   **Caching:** 2-second in-memory cache.

#### `GET /ai-insights`
Returns GenAI situational analysis generated by **Google Gemini 2.5 Flash**.
*   **Caching:** 30-second logic-level cache.

#### `GET /queue`
Returns all food stalls with dynamically calculated wait times based on their zone's current density.

---

### 🔵 Fan Interaction

#### `POST /route`
Calculates the density-weighted optimal path between any two zones using Dijkstra's algorithm.
*   **Body:** `{ "from": "A", "to": "D" }`
*   **Response:** Includes path coordinates, estimated minutes, and reward points.

#### `POST /best-route`
Automatically finds the quietest reachable zone from the fan's current location and provides a path.
*   **Body:** `{ "from": "C" }`

#### `POST /order`
Places a simulated food order.
*   **Body:** `{ "stallId": "s1", "items": ["Hot Dog", "Cola"], "userId": "fan-123" }`
*   **Rate Limit:** 10 requests per minute.

---

### 🔴 Operational Controls

#### `POST /simulate` (Admin)
Changes the stadium simulation mode.
*   **Header:** `X-Admin-Key: <your-key>`
*   **Body:** `{ "mode": "halftime" }`
*   **Modes:** `normal`, `pre_match`, `halftime`, `exit_rush`.

#### `POST /alert`
Allows staff members to manually inject high-priority alerts into the system.
*   **Body:** `{ "zone": "B", "message": "Spill near exit", "type": "warning" }`

---

## ⚡ WebSocket Interface (Socket.IO)

The system broadcasts live telemetry every 2 seconds to all connected clients.

### 📥 Server-to-Client Events

#### `simulation_update`
Emitted every 2 seconds. Contains the complete state of the stadium.
```json
{
  "density": { "A": 0.45, "B": 0.82, ... },
  "alerts": [...],
  "metrics": { "avgDensity": 0.41, "congestionReduced": 35, ... },
  "mode": "normal",
  "tick": 425
}
```

#### `mode_change`
Emitted when an admin updates the simulation goal.

#### `proactive_nudge`
Emitted when a critical density spike is detected, triggering a push notification to affected fans.

---

## 🛡️ Response Codes

| Code | Meaning |
|---|---|
| `200` | Success |
| `400` | Bad Request (invalid zone ID, malformed JSON) |
| `401` | Unauthorized (missing/wrong X-Admin-Key) |
| `429` | Rate Limit Exceeded |
| `500` | Internal Server Error |
